Fuzzy Controls Simulation

Run the Fuzzy_Control.mat file in MATLAB to see the simulation. Red dot is the target, blue triangle is the vehicle/car, light blue is the trajectory of the car.

This code shows an example of fuzzy logic controls (mamdani method) of a, say, self-driving car. It controls the movement response of the car, adjusting as necessary depending on how far the car is from the target. The farther it is, the faster the car will drive in order to catch up.

The fuzzy logic controller is also included in the repository with the name Control_new.fis. To edit, call ‘fuzzy’ in the MATLAB command line to bring up the GUI, and import the .fis file.

Fuzzy logic itself is an intuitive way of defining the behaviours of a controller. In this situation for example, the car has to chase after a moving target. Instead of having two modes only: driving and stopping, with fuzzy control we can also have ‘drive slow’ and ‘drive fast’ modes, or even ‘drive slowest’, ‘drive slow’, ‘drive fast’, ‘drive faster’, ‘drive even faster’, even ‘like there’s no morrow’.

The control system was generated using MATLAB's Fuzzy Logic Toolbox. This takes into account four things: distance input, angular distance input, velocity output, and angular velocity output.

Change the membership functions of each one by double-clicking on any one of the four boxes.

Notice that the mf allocations of Distance and Angular Distance inputs are similar, differing only in the scale. They’re just for convenience, feel free to change them as needed.

Here is where we define ‘how far is far?’ and vice-versa. You can set the range, name of the membership function (whose relationships will need to be defined for each one of them). In this example I’ve set all of the mfs to be triangles (trimf… tri-mf) again for convenience’s sake – change as needed.

The velocity output here is set up so that it doesn’t have any negative-value response for output. This is because I didn’t want the car to reverse, and since negative output on the velocity is used to make the car go backwards the controller has no use of it in this case.

With the angular velocity though, I wanted to have the option to turn in the opposite direction – left or right, so I added mfs for both cases.

The inputs-outputs are not linked automatically; they have to be defined using IF… THEN… format for each case. Just adding more mfs to the plot won’t make a difference if the new mfs’ relationships aren’t defined in the rules.

Edit -> Rules to bring up the Rules tab.

You can change the rules in the Rules tab. Note that the rules have to be defined manually for each case or the controller will just ignore them (not taken into account).

After editing, export into file/workspace. If you exported it as a .fis file you’d need to change the Fuzzy_Control.mat file on the line
fis = readfis('Your_new_fis_here.fis’);.
